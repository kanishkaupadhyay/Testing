## 🧪 4. More Testing Types

### 31. What is Black Box Testing?

No knowledge of internal code.

### 32. What is White Box Testing?

Testing internal logic.

### 33. What is Grey Box Testing?

Combination of both.

### 34. What is End-to-End Testing?

Testing full workflow.

### 35. What is Exploratory Testing?

Ad-hoc testing without predefined cases.

### 36. What is Ad-hoc Testing?

Random testing.

### 37. What is Performance Testing?

Checks speed and stability.

### 38. What is Load Testing?

Expected load testing.

### 39. What is Stress Testing?

Beyond limits.

### 40. What is Scalability Testing?

System growth handling.

## 🐞 5. Bug Lifecycle

### 41. What is a Bug?

A defect in the software.

### 42. Bug Lifecycle stages?

New → Assigned → Open → Fixed → Retest → Closed

### 43. What is Severity?

Impact of bug.

### 44. What is Priority?

Urgency of fix.

### 45. Severity vs Priority?

- Severity → impact  
- Priority → urgency  

### 46. What is Critical Bug?

System crash.

### 47. What is Blocker Bug?

Stops testing.

### 48. What is Minor Bug?

Low impact.

### 49. What is Reopened Bug?

Bug still exists after fix.

### 50. What is Deferred Bug?

Postponed.

## 🧾 6. Defect Management

### 51. What is Defect Report?

Bug description document.

### 52. Fields in defect report?

- ID  
- Description  
- Steps  
- Severity  
- Status  

### 53. What is Root Cause Analysis?

Finding reason of bug.

### 54. What is Bug Triage?

Prioritizing bugs.

### 55. Who assigns bug priority?

Product owner/manager.
