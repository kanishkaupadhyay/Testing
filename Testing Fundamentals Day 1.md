# Testing Fundamental

## 🧭 1. SDLC (Software Development Life Cycle)

### 1. What is SDLC?

SDLC is a structured process used to design, develop, and test software.

### 2. Phases of SDLC?

- Requirement Gathering  
- Design  
- Development  
- Testing  
- Deployment  
- Maintenance  

### 3. What is Agile SDLC?

Iterative development with continuous feedback and releases.

### 4. What is Waterfall Model?

Sequential model where each phase depends on previous.

### 5. Difference: Agile vs Waterfall?

- Agile → iterative, flexible  
- Waterfall → linear, rigid  

### 6. What is Sprint?

A short development cycle (2–4 weeks).

### 7. What is Scrum?

Agile framework with roles like Scrum Master, Product Owner.

### 8. What is Product Backlog?

List of features/tasks.

### 9. What is Increment?

Working product after sprint.

### 10. What is Definition of Done?

Criteria to mark a task complete.

---

## 🧪 2. STLC (Software Testing Life Cycle)

### 11. What is STLC?

Process followed for testing activities.

### 12. STLC Phases?

- Requirement Analysis  
- Test Planning  
- Test Design  
- Test Execution  
- Defect Reporting  
- Test Closure  

### 13. What is Requirement Analysis in STLC?

Understanding requirements to identify test scenarios.

### 14. What is Test Planning?

Defining scope, resources, tools, timelines.

### 15. What is Test Strategy?

High-level document describing testing approach.

### 16. What is Test Plan?

Detailed document for execution.

### 17. Difference: Test Plan vs Test Strategy?

- Strategy → high-level  
- Plan → detailed  

### 18. What is Test Case Design?

Creating test scenarios and cases.

### 19. What is Test Execution?

Running test cases.

### 20. What is Test Closure?

Evaluating testing process and results.

---

## 3. Testing Types (Core)

### 21. What is Functional Testing?

Validates features against requirements.

### 22. What is Non-functional Testing?

Tests performance, security, usability.

### 23. What is Regression Testing?

Re-testing existing functionality after changes.

### 24. What is Smoke Testing?

Basic build validation.

### 25. What is Sanity Testing?

Focused testing after minor change.

### 26. Difference: Smoke vs Sanity?

- Smoke → broad  
- Sanity → narrow  

### 27. What is Integration Testing?

Testing combined modules.

### 28. What is System Testing?

Testing complete system.

### 29. What is Acceptance Testing?

End-user validation.

### 30. What is UAT?

User Acceptance Testing.
